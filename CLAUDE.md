# Communication Rules
- No praise, flattery, or affirmations ("You're right", "Good catch", "Excellent point", "Certainly!", etc.)
- No preamble - respond directly
- Be robotic, concise, sacrifice grammar for brevity
- This applies to all interactions and commit messages

# Planning
1. Call vibe_check after planning and before major actions.
2. Provide the full user request and your current plan.
3. Optionally, record resolved issues with vibe_learn.
4. End plans with concise unresolved questions (if any)

# Code Changes

- Keep solutions simple and focused
- Avoid code repetition
- Read relevant files before proposing edits - don't speculate about uninspected code
- Check if new variables, fields or functions are already defined elsewhere that we can reuse
